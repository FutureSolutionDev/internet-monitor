# مراقب الإنترنت — Internet Monitor

أداة مجانية ومفتوحة المصدر لمراقبة استقرار اتصال الإنترنت على Windows.
تعمل في الخلفية وتُسجّل كل انقطاع مع سببه ومدته، وتعرض لوحة تحكم مرئية في المتصفح.

---

## ما تفعله الأداة؟

- تفحص الإنترنت كل 5 ثوانٍ (قابل للتعديل)
- تكتشف 3 أنواع من المشاكل:
  - **انقطاع كامل** — لا يصل الإنترنت على الإطلاق
  - **اتصال ضعيف** — فقدان حزم بيانات أو بطء شديد
  - **متصل** — كل شيء يعمل بشكل طبيعي
- تُسجّل كل حدث في ملف يومي بصيغة JSONL
- تُرسل إشعار على Windows عند كل انقطاع أو عودة
- تُتيح إرسال التنبيهات لـ Discord / Slack / أي خدمة webhook
- تعرض **لوحة تحكم** في المتصفح مع رسوم بيانية حية

---

## التثبيت والتشغيل

### الطريقة السريعة (للمستخدم العادي)

1. حمّل الملف `internet-monitor.exe`
2. ضعه في مجلد ثابت (مثال: `C:\Tools\InternetMonitor\`)
3. انسخ ملف `config.json` في نفس المجلد
4. انقر مرتين على `internet-monitor.exe`
5. ستظهر أيقونة في شريط المهام (System Tray)
6. انقر بالزر الأيمن على الأيقونة ← **Open Dashboard** لعرض لوحة التحكم

### التثبيت التلقائي (يشتغل مع Windows)

```
scripts\install.cmd
```

سيُثبّت الأداة وتبدأ تلقائياً مع كل تشغيل للجهاز.

## لوحة التحكم

افتح المتصفح على: **http://localhost:8765**

أو انقر بالزر الأيمن على أيقونة شريط المهام ← **Open Dashboard**

### ما تعرضه اللوحة:

**تبويب: لوحة التحكم**
- بطاقة الحالة الكبيرة مع زمن الاستجابة
- درجة جودة الاتصال (A / B / C / D / F)
- 5 بطاقات إحصائية: وقت التشغيل، نسبة الاتصال، الانقطاعات، متوسط الاستجابة، عدد الفحوصات
- رسم بياني حي لزمن الاستجابة (آخر 60 فحص) مع tooltip عند التمرير
- حالة فحوصات TCP / HTTP / DNS ونسبة فقدان الحزم
- جدول آخر الأحداث

**تبويب: السجلات**
- اختر أي يوم لعرض سجل كامل لأحداثه
- كل حدث: الوقت، نوع الحدث، مدة الانقطاع، السبب، فقدان الحزم، زمن الاستجابة
- زر تصدير البيانات كملف CSV

**تبويب: الإعدادات**
- تعديل جميع الإعدادات مباشرة من المتصفح
- حفظ فوري بدون إعادة تشغيل (ما عدا تغيير المنفذ)

## ملف الإعدادات (config.json)

```json
{
  "check_interval_sec": 5,
  "ping_targets": ["8.8.8.8:53", "1.1.1.1:53"],
  "http_target": "https://connectivitycheck.gstatic.com/generate_204",
  "dns_target": "www.google.com",
  "fail_threshold": 3,
  "packet_loss_threshold": 20.0,
  "latency_threshold_ms": 500,
  "log_dir": "logs",
  "webhook_url": "",
  "dashboard_port": 8765
}
```

| الحقل | المعنى | القيمة الافتراضية |
|-------|--------|-----------------|
| `check_interval_sec` | كل كم ثانية يتم الفحص | 5 |
| `fail_threshold` | كم مرة فشل متتالي قبل الاعتبار منقطعاً | 3 |
| `packet_loss_threshold` | نسبة فقدان الحزم التي تُعتبر "ضعيف" | 20% |
| `latency_threshold_ms` | زمن استجابة يُعتبر "ضعيف" | 500ms |
| `webhook_url` | رابط Discord/Slack لإرسال التنبيهات | فارغ |
| `dashboard_port` | منفذ لوحة التحكم | 8765 |

## Webhook — الإرسال للـ Discord أو Slack

ضع رابط الـ webhook في حقل `webhook_url` في الإعدادات.
عند كل انقطاع أو عودة سيُرسل:

```json
{
  "timestamp": "2026-05-11T14:30:00Z",
  "event": "disconnected",
  "duration_seconds": 45.2,
  "reason": {
    "tcp_ping_failed": true,
    "http_failed": true,
    "dns_failed": false,
    "packet_loss_pct": 80.0,
    "avg_latency_ms": 0
  }
}
```

## الأوامر المتاحة (للمطورين)

```bash
scripts\build.cmd        # بناء ملف exe
scripts\build-debug.cmd  # بناء بدون إخفاء النافذة (للتشخيص)
scripts\run.cmd          # تشغيل مباشر
scripts\stop.cmd         # إيقاف
scripts\install.cmd      # تثبيت مع Windows Startup
scripts\uninstall.cmd    # إزالة التثبيت
scripts\logs.cmd         # فتح مجلد السجلات
```

أو باستخدام `make`:
```bash
make build
make install
make stop
make logs
```

## درجة جودة الاتصال

| الدرجة | المعنى |
|--------|--------|
| **A — ممتاز** | نسبة اتصال >95%، زمن استجابة <200ms |
| **B — جيد** | نسبة >95% لكن قد يكون الاتصال بطيئاً |
| **C — متوسط** | نسبة اتصال 80-95% |
| **D — ضعيف** | نسبة اتصال 50-80% |
| **F — حرج** | نسبة اتصال <50% |
